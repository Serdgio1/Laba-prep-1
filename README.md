# Лабораторная работа: "Инженер АвтоМир — Разработка модулей для современного автомобиля"

Вы — разработчик программного обеспечения для концепта "умного автомобиля" AutoМир. Вам поручено реализовать "сердце" систем управления автомобилем: коллекции и обработку датчиков, компонентов и абстракцию управления частями авто. Задание предполагает реализацию низкоуровневых контейнеров и управление ресурсами как на "настоящем железе".

---

## Техническое задание

### 1. **Своя реализация вектора (MyVector)**
- Класс-шаблон собственного динамического массива (вектора).
- Должен поддерживать: хранение любого типа, push_back, pop_back, operator[], size, capacity, ручное управление памятью.
- Реализовать **правило 5** (конструктор копирования, оператор копирующего присваивания, деструктор, перемещающий конструктор, оператор перемещающего присваивания).

### 2. **Своя реализация unique_ptr (MyUniquePtr)**
- Класс-шаблон, который реализует уникальное владение динамическим объектом с семантикой перемещения ("unique ownership", один владелец; копирование запрещено).
- Обязательные функции: конструктор, деструктор, operator*, operator->, метод release(), метод reset(), перемещающее присваивание.

### 3. **Связный список (MyLinkedList)**
- Класс-шаблон, реализующий односвязный список (Node).
- Поддержка методов: push_front, push_back, pop_front, pop_back, operator[], size(), clear().
- Обеспечить правильное освобождение памяти.

---

## Архитектурные требования

- Реализовать абстрактный класс **AutoComponent** с чисто виртуальной функцией `void diagnose() = 0`.
- На базе AutoComponent создать конкретные классы: `Engine`, `Sensor`, `Transmission`, каждый реализует diagnose по-своему.
- В тестовом main.cpp создать MyVector<MyUniquePtr<AutoComponent>>, наполнить разными компонентами авто (через new и MyUniquePtr), выводить диагностику по всем элементам вектора:  
  Для каждого объекта вызвать виртуальную функцию `diagnose()`.

- Должна быть хотя бы одна функция, где используется **абстрактный класс и виртуальные методы с контейнером** (см. выше).
- Продемонстрировать корректную работу "правила 5" для одного из ваших контейнеров.
